This API updates dynamic attributes in SSM.
The `Authorization` must have `Bearer` followed by `Token`.
Mandatory params:
`securitysystem`
`endpoint`
`updateuser`
`dynamicattributes` - `attributename`
Optional params:
- `dynamicattributes` - other attributes of dynamic attributes - `attributetype`(BOOLEAN/NUMBER/STRING/ENUM/MULTIPLE SELECT FROM LIST/MULTIPLE SELECT FROM SQL QUERY/SINGLE SELECT FROM SQL QUERY/PASSWORD/LARGE TEXT/CHECK BOX/DATE) - `requesttype`(ACCOUNT/PROPOSED ENTITLEMENT OWNER/ENTITLEMENT OWNER ATTESTATION/ROLE/SERVICE ACCOUNT) - `attributelable`: Specify the name of the attribute, which you want to display in the Access Requests request form.
