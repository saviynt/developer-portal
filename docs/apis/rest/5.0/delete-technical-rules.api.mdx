Starting with Release v23.11, you can use the REST API to delete an inactive technical rules from the target system by specifying the `ruleId` parameter.

**Note:**

- You can only delete an inactive technical rules.
- You cannot delete technical rules if:

    - the Rule is active.

    - the ID does not exist.

    - the user who perform the delete process is not a ROLE_ADMIN.


The `Authorization` must have `Bearer` followed by `Token`.

Mandatory param:

`ruleId`
